    LLMProvider¶

    wordbase.provider ¶

    Abstract base class for LLM providers.

    This module defines the LLMProvider abstract base class that all provider implementations must inherit from. The provider interface ensures a consistent API across different LLM backends (HuggingFace, OpenAI, Anthropic, etc.).

    Key responsibilities of providers
    • Convert between WordBase's internal message format and provider APIs
    • Make API calls or run local inference
    • Format responses into WordBase's standard completion format
    • Handle caching of responses
    • Support batch processing

    Typical usage example:

    from wordbase.providers.huggingface import HuggingFaceProvider
from wordbase.types import Message
from wordbase.cache import MemoryCache

provider = HuggingFaceProvider(model_name="gpt2")
provider.set_cache(MemoryCache())

messages = [Message(role="user", content="Hello")]
completions = provider.generate(messages, temperature=0.7)

    """Generate completions given conversation history.

    Args:
        messages: Conversation history as list of Message objects.
        system_prompt: System prompt to prepend (if provided).
        n: Number of completions to generate.
        max_tokens: Maximum tokens per completion.
        temperature: Sampling temperature (0 = greedy decoding).
        top_p: Nucleus sampling threshold.
        top_k: Top-k sampling (provider-dependent).
        num_beams: Beam search width (1 = no beam search).
        stop_sequences: Stop generation at these strings.
        stream: Enable streaming mode (yields partial results).
        use_cache: Whether to use cache for this request (default: True).
        return_token_probs: Whether to return token probabilities.
        logprob_aggregation: How to aggregate token logprobs into text logprob.
            Options: "sum", "arithmetic_mean" (default: "arithmetic_mean").
        **kwargs: Provider-specific parameters.

    Returns:
        List of completion dictionaries (length n), each containing:
            - text: Full completion string (excludes thinking block if present)
            - thinking: Reasoning content from <think>...</think> block,
                        only present when the model emits thinking tokens
            - logprob: Aggregated log probability (when return_token_probs=True)
            - tokens: List of token dicts with 'text', 'logprob', 'token_id'
                      (when return_token_probs=True)
            - finish_reason: Why generation stopped ('stop', 'length', etc.)
            - metadata: Provider-specific information
    """
